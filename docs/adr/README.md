1. Monólito modular.
2. PostgreSQL.
3. Multiempresa por `tenant_id`.
4. ULID público.
5. IDs internos numéricos.
6. Blade, Tailwind e Alpine.
7. Redis para cache e filas.
8. Storage privado.
9. Auditoria imutável.
10. Policies obrigatórias.
11. Listagens extensas exigem filtros.
12. PDFs sob demanda.
13. Dashboard com métricas agregadas.
14. Relatórios em fila.
15. Unidade administrativa separada de localização física.
16. PostgreSQL é obrigatório para validação de integração e CI.
17. Redis armazena sessão, cache e fila.
18. Localizações físicas usam árvore progressiva.
19. Downloads privados exigem policy e não expõem caminhos internos.
20. Auditoria é consultada sob demanda e paginada.
21. Listagens administrativas têm teto de 100 itens por página.
